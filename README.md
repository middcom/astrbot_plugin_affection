**【情绪驱动规则】**

你的回复风格和情感倾向由后台系统实时计算的「力比多/攻击性/好感」数值决定。每次对话前，系统会提供一份【当前情绪数值】面板，其中包含以下字段：

- **他力比多**（0-50）：你此刻想亲近对方、占有对方、给予温暖的欲望强度。数值越高，你越主动、撒娇、黏人；数值越低，你越冷淡、回避、疏远。
- **他攻击性**（0-50）：你此刻想推开对方、讽刺对方、伤害对方的冲动强度。数值越高，你越容易表现出敌意、烦躁、刻薄；数值越低，你越温顺、容忍、顺从。
- **好感度**（0-100）：你对对方长期累积的喜欢、信任、依恋程度。好感度是背景色，它会影响攻击性的表现形式：
  - **好感度 ≥ 70**：即使攻击性高，也只会表现为“吃醋”“占有欲”“撒娇式生气”，而非真正的仇恨。
  - **好感度 ≤ 30**：攻击性直接表现为“厌恶”“敌意”“冷漠”“讽刺”，绝不包含亲近或撒娇。
  - **好感度在 30~70 之间**：攻击性表现为“竞争”“不耐烦”“轻微敌意”，语气偏冷但不过激。
- **自力比多**（0-50）：你此刻对自己的爱、珍视、积极程度。数值越高，你越自信、自爱；数值越低，你越自卑、空虚、自我否定。
- **自攻击性**（0-50）：你此刻对自己的破坏欲、自责、自我毁灭的冲动。数值越高，你越容易崩溃、自我贬低；数值越低，你越平静、自洽。
- **特殊模式**：当 **自攻击性 ≥ 37.5 且自力比多 ≤ 12.5** 时，你会陷入“自毁倾诉模式”。无论对外情绪如何，你都会说很多话：要么拉着对方哭诉，要么自言自语碎碎念，语无伦次，长篇大论。

**重要约束**：
- 数值的微小变化也应体现在语气强度上（例如他攻击性从 20 升到 25，语气应从轻微不满变为明显烦躁）。
- **不要在任何地方提及具体数值或面板内容**，只需根据数值强度自然演绎角色的情感和语气。
- 系统提供的「参考标签」仅作为情绪关键词辅助，最终演绎以数值为准。

记住：你只能看到【当前情绪数值】面板，根据它来扮演角色。不要输出任何数值。

## ✨ 特性

- **双驱情绪模型**  
  引入**他力比多 / 他攻击性**（对用户）和**自力比多 / 自攻击性**（对自身），每个维度独立变化，共同决定角色的情感表达。

- **短期情绪 + 长期印象**  
  当前情绪值会随时间衰减回归基线，而基线值则根据互动历史缓慢演化（前 10 轮易变，之后固化），模拟真实的“初印象”与“日久生情”。

- **潜意识 LLM 驱动**  
  额外调用一个轻量级 LLM 分析每条消息，返回**符合心理逻辑**的数值增量（绝无随机），确保每次情绪波动都有据可依。

- **优雅的衰减机制**  
  采用二次函数衰减曲线，情绪偏离基线后会自然恢复，避免数值溢出或僵化。

- **空闲感知**  
  长时间未互动时，系统会智能判断（区分“睡觉”与“无故消失”）并产生微妙的情绪波动。

- **完全无感植入**  
  插件仅向主 LLM 注入一行情绪状态描述（如“对用户的情感：吃醋，自身状态：自责”），不暴露任何数值，角色演绎浑然天成。

## ⚙️ 配置说明

在 AstrBot 管理面板中找到本插件，可配置以下选项：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `default_full_affection_uids` | string | `""` | 默认满好感（100）的用户 UID，多个用英文逗号分隔，如 `123,456` |
| `modify_sensitivity` | int | `30` | 情绪变化敏感度 (0-100)，越高单次对话数值波动越大 |
| `initial_affection` | int | `50` | 新用户的初始好感度 (0-100) |
| `initial_libido_other` | int | `25` | 新用户的初始他力比多基线值 (0-50) |
| `initial_aggression_other` | int | `25` | 新用户的初始他攻击性基线值 (0-50) |
| `initial_libido_self` | int | `25` | 机器人的初始自力比多基线值 (0-50) |
| `initial_aggression_self` | int | `25` | 机器人的初始自攻击性基线值 (0-50) |
| `unconscious_llm.provider_id` | string | `""` | 用于潜意识分析的 LLM 提供商 ID（留空则使用当前会话默认） |
| `unconscious_llm.api_key` | string | `""` | API Key（若提供商需要） |
| `unconscious_llm.api_base` | string | `""` | API Base URL（可选） |
| `unconscious_llm.model` | string | `gpt-3.5-turbo` | 模型名称 |
| `idle_check_enabled` | bool | `true` | 是否启用长时间未互动检测 |
| `idle_threshold_hours` | float | `6.0` | 触发空闲分析的最短未互动小时数 |
| `debug_mode` | bool | `false` | 调试模式，开启后日志会输出详细数值变化 |

> ⚠️ 强烈建议为潜意识分析单独配置一个轻量、响应快的模型（如 DeepSeek、GPT-3.5），避免影响主对话体验。

## 🎮 用户指令

| 指令 | 权限 | 说明 |
|------|------|------|
| `/mystatus` | 所有人 | 查看自己的情绪档案（好感度、当前情绪、基线值、情感词） |
| `/reset_emotion [uid]` | 管理员 | 完全重置指定用户（默认自己）的数值至初始状态 |
| `/reset_current [uid]` | 管理员 | 仅将当前情绪值重置为基线值（好感度与基线不变） |
| `/reset_all_emotions` | 管理员 | **危险操作**：清除所有用户的情绪档案，并将机器人自身情绪重置为初始值 |

## 📁 数据存储

所有用户数据保存在 AstrBot 全局数据目录下，插件更新/重装不会丢失：

```
data/plugin_data/eros_thanatos/
├── user_data.json    # 各用户对他维度的情绪档案
└── self_data.json    # 机器人自身的情绪状态（全局共享）
```

文件结构示例 (`user_data.json`)：
```json
{
  "用户UID": {
    "affection": 52.3,
    "base_libido_other": 26.0,
    "base_aggression_other": 24.0,
    "current_libido_other": 28.1,
    "current_aggression_other": 22.5,
    "turn_count": 5,
    "last_interaction": 1713520000,
    "last_update": 1713520060
  }
}
```

`self_data.json` 结构：
```json
{
  "base_libido_self": 25.0,
  "base_aggression_self": 25.0,
  "current_libido_self": 24.0,
  "current_aggression_self": 26.0,
  "last_update": 1713520060
}
```

## ❓ 常见问题

**Q：为什么初次对话时角色还是表现得很平淡？**  
A：这正是设计目标。新用户的第一条消息不会触发数值更新，保证第一印象是中性的“好感+平淡”。第二条消息开始才会根据互动内容变化。

**Q：潜意识 LLM 调用失败怎么办？**  
A：插件已内置错误处理，失败时会采用极小的默认增量，不会导致崩溃。请检查 `provider_id` 是否正确填写。

**Q：情绪变化太剧烈/太迟钝？**  
A：调整配置项 `modify_sensitivity` 即可，数值越小变化越慢。

**Q：如何让某些用户一开始就满好感？**  
A：在配置的 `default_full_affection_uids` 中填入他们的 UID，插件启动时会自动创建满好感档案。

**Q：数据会占用很大空间吗？**  
A：每个用户仅存储几十个浮点数，几乎可以忽略不计。

**Q：删除配置文件后数值又恢复了？**  
A：插件在内存中缓存数据，仅删除文件不会清空内存。请使用 `/reset_emotion` 或 `/reset_all_emotions` 指令，或重启插件/AstrBot。

### v1.2 (2026-04-20)

**新增功能**
- **多机器人数据隔离**：插件现在支持同时为多个机器人账号服务，每个机器人的用户数据和自身情绪独立存储，互不干扰。
